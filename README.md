# spring-cloud

## MSA 설계 원칙

- 자율적인 기능 
- 배포 단위의 크기
- 분리하기에 가장 적합한 기능 또는 서브 도메인 
- 폴리글랏 아키텍쳐
- 선택적 확장
- 애자일 팀과 협력 개발
- 단일 책임
- 복제 가능성과 변경 가능성
- 결합과 응집
- 마이크로서비스를 하나의 제품으로 생각하기

## 스프링 클라우드, 클라우드 파운드리 차이점
- spring boot application 을 만드는 데 사용되는 개발자 키트
- 클라우드 파운드리는 애플리케이션의 개발, 배포, 확장을 위한 오픈소스 PaaS 플랫폼
